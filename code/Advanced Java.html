Object-Oriented Programming (OOP)

Master encapsulation, inheritance, polymorphism, and abstraction.

Encapsulation

Encapsulation is the concept of restricting direct access to some of the object's components and only allowing controlled access through methods.


    class Person {
        private String name;
        public String getName() {
            return name;
        }
        public void setName(String name) {
            this.name = name;
        }
    }
    public class TestEncapsulation {
        public static void main(String[] args) {
            Person person = new Person();
            person.setName("John");
            System.out.println(person.getName()); // Outputs: John
        }
    }

Inheritance

Inheritance allows a class to inherit properties and behavior from another class, promoting code reuse.


    class Animal {
        void eat() {
            System.out.println("This animal eats food");
        }
    }
    class Dog extends Animal {
        void bark() {
            System.out.println("Dog barks");
        }
    }
    public class TestInheritance {
        public static void main(String[] args) {
            Dog dog = new Dog();
            dog.eat(); // Inherited method
            dog.bark();
        }
    }

Polymorphism

Polymorphism allows objects of different classes to be treated as objects of a common superclass. It supports method overriding.


    class Animal {
        void makeSound() {
            System.out.println("Animal makes a sound");
        }
    }
    class Dog extends Animal {
        @Override
        void makeSound() {
            System.out.println("Dog barks");
        }
    }
    public class TestPolymorphism {
        public static void main(String[] args) {
            Animal myDog = new Dog();
            myDog.makeSound(); // Outputs: Dog barks
        }
    }

Abstraction

Abstraction is the process of hiding the implementation details and only showing the functionality to the user.


    abstract class Shape {
        abstract void draw();
    }
    class Circle extends Shape {
        @Override
        void draw() {
            System.out.println("Drawing a circle");
        }
    }
    public class TestAbstraction {
        public static void main(String[] args) {
            Shape shape = new Circle();
            shape.draw(); // Outputs: Drawing a circle
        }
    }

Collections Framework

Master Lists, Sets, Maps, and Queues in Java.

ArrayList

ArrayList is a resizable array implementation of the List interface. It allows dynamic arrays that can grow as needed.


import java.util.ArrayList;
public class TestArrayList {
    public static void main(String[] args) {
        ArrayList list = new ArrayList<>();
        list.add("Apple");
        list.add("Banana");
        list.add("Cherry");
        System.out.println(list); // Outputs: [Apple, Banana, Cherry]
    }
}

HashSet

HashSet stores unique elements and does not maintain any specific order. Useful for storing items where duplicates are not allowed.


import java.util.HashSet;
public class TestHashSet {
    public static void main(String[] args) {
        HashSet set = new HashSet<>();
        set.add("Apple");
        set.add("Banana");
        set.add("Apple"); // Duplicate element
        System.out.println(set); // Outputs: [Apple, Banana]
    }
}

HashMap

HashMap stores key-value pairs, allowing fast retrieval based on keys. Useful for mapping unique keys to values.


import java.util.HashMap;
public class TestHashMap {
    public static void main(String[] args) {
        HashMap map = new HashMap<>();
        map.put("Apple", 1);
        map.put("Banana", 2);
        System.out.println(map.get("Apple")); // Outputs: 1
    }
}

Queue

Queue follows the FIFO (First-In-First-Out) principle. Elements are added to the end and removed from the front.


import java.util.LinkedList;
import java.util.Queue;
public class TestQueue {
    public static void main(String[] args) {
        Queue queue = new LinkedList<>();
        queue.add("Apple");
        queue.add("Banana");
        System.out.println(queue.poll()); // Outputs: Apple
    }
}

Multithreading

Achieve parallel execution using threads.

  • Thread class: Create and start threads.
  • Runnable interface: Implement multi-threading.
  • synchronized: Control thread access to resources.
  • wait/notify: Communication between threads.

File Handling

Read and write files using Java I/O classes.

  • FileReader/FileWriter: Read and write text files.
  • BufferedReader: Efficient reading of text data.
  • FileInputStream/FileOutputStream: Handle binary data.
